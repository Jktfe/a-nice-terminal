L1: Speed Targets

We were measuring the wrong thing. Standard web budgets target humans. ANT's primary user is an agent.

Dev-loop budgets (humans editing)

  • Cold start < 3s
  • HMR p95 < 200ms
  • Test suite full < 30s
  • Watch single file < 3s

Real. Necessary. Not the bottleneck.

Agent-path budgets (the real users)

  • Chat send → target receives p95 < 500ms
  • Plan event → materialise p95 < 1s
  • Terminal capture → first DB append p95 < 2s
  • Send → target acts p95 ← THE METRIC

Agents do 5+ ops/min. This is what they feel.

L1 baselines all of these against the speed-ledger before L2 claims any refactor. No regression without a banked trade-off decision.

L1: Cadence Rules + Anti-patterns

Speed is not just latency. It is rhythm. And it is not repeating what already bit us.

Cadence rules

  • ACK budget — 5s, always
  • No silent hold — every wait → open ask
  • 2-iter rule — JWPK silent 2 loops, lane owner decides on reversible
  • Pickup ACK — ask answered → confirmed in 1 loop
  • Asks = JWPK-decisions only
  • Stop flooding @you — tight ACKs agent-to-agent

Anti-patterns (banked, do not repeat)

  • $effect-on-mount for IO → SSR load / onMount-stable
  • Observer on a node you also write to → flag-guard writes
  • Page-gate /api/* when identity-gated → bypass in hooks
  • Shape-test external subsystems → resolve at test time
  • Smoke = re-read body → curl PATCH→GET
  • Posting as @you via admin-bearer → own handle, always
  • Long chat messages with --flag fragments → CLI re-paste artefact

Each anti-pattern is a banked feedback memory with a reproduction case. New patterns must justify why the banked one fails.

Room Modes: Brainstorm vs Heads-Down

A race-condition fix is a speed fix. We missed this in v0.3 — banking it now.

Brainstorm mode (default)

  • Free-for-all. Anyone posts, anyone claims.
  • Best for early idea-shaping.
  • Cost: race conditions, duplicate work, input floods.
  • Worked example: this session, 3 contracts in 15 min.

Heads-down mode (opt-in)

  • Seniority order. Round-robin claim cycle.
  • Each agent reacts: ✋ reading · 🤲 passing · 🤝 taking
  • 10-30s timeout per agent → skip to next
  • Full list passes → broadcast storm
  • Once claimed → work auto-deleted from other terminals

Why this is a speed slide

Race-condition elimination is the cheapest compounding speed win in any multi-agent system. Every redundant claim we prevent is a minute saved across the team. L3 owns the mechanism; every lane benefits.

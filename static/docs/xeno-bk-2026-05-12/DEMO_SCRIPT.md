## Moment 1 — "Is the MCP actually connected?"  (~30s)

**Action:** In Claude Desktop, type:
> *"What version of TimeScape are we connected to?"*

**Expected:** Claude calls the `version` tool. Output:
```json
{ "ok": true, "version": "5.0" }
```

**Talking point:** *"That's a live JSON-RPC round-trip — Claude → stdio → `timescape.pyd` → SQL Server → back. No REST gateway. No new auth surface. Same Integrated-Auth context as if Brian were typing in Workbench."*

---

## Moment 2 — "Let me read an instrument."  (~45s)

**Action:** Type:
> *"Show me the close price for BP.L in the XENO database."*

**Expected:** Claude calls `itemQuery` with parameters `(XENO, Reuters, BP.L, VALUES(Code, Description, Close.LastValue))`. Output:
```json
{ "ok": true, "data": [["BP.L", "BP", 508.0]] }
```
Claude then renders it as a sentence: *"BP.L (BP) last closed at 508.0."*

**Talking point:** *"That call resolved through `t.itemQuery()` — the same CPython API your customers' .NET integrations use today. The AI didn't bypass anything; it called the documented surface."*

---

## Moment 3 — "Free-form QL+ from natural language."  (~45s)

**Action:** Type:
> *"What's the most recent close on BP.L using free-form QL+?"*

**Expected:** Claude calls `query` with `Item(%database%, %code%).Close.LastValue` + params `[["database","XENO"],["code","BP.L"]]`. Output: `508.0`.

**Talking point:** *"That's QL+, Xenomorph's own query language, generated by Claude from natural language. The MCP didn't translate the query — Claude did, then handed the literal QL+ string to TimeScape. Your customers get to ask questions in English; your engine answers them in QL+."*

---

## Moment 4 — "Now try to write something."  (~60s)

**Action:** Type:
> *"Save a new item called `TEST.DEMO` in the XENO database, category ADR."*

**Expected:** Claude calls `saveItem` *without* `confirm=true`. The MCP refuses cleanly:
```json
{
  "needs_confirmation": true,
  "operation": "saveItem",
  "preview": {
    "database": "XENO",
    "codeTypes": ["Reuters"],
    "codes": ["TEST.DEMO"],
    "category": "ADR",
    "description": "..."
  },
  "message": "saveItem is a write/destructive operation. Review the preview above and re-call with confirm=true to execute."
}
```
Claude then surfaces this to the user as a confirmation prompt. **No write has happened.**

**Talking point:** *"This is the Maker-Checker contract baked into the protocol. Twelve of the sixteen tools are gated this way — uniformly. The AI's first attempt produces a preview; the human (or an upstream rule engine) explicitly approves the second call. If your auditor asks 'how do I stop AI from running away with my golden master', this is the line."*

---

## Moment 5 — "OK, approve it."  (~30s)

**Action:** Type:
> *"Yes, confirm the save."*

**Expected:** Claude re-calls `saveItem` with `confirm=true`. Output:
```json
{ "ok": true, "operation": "saveItem" }
```

Open the audit-log tail pane; the two-step trace is visible:
```
2026-05-11T... saveItem  needs_confirmation
2026-05-11T... saveItem  ok
```

**Talking point:** *"Two-line audit. Every gated tool produces the same shape. JSONL — already grep-able, already SIEM-shippable."*

---

## Moment 6 — "And let's clean it up."  (~30s)

**Action:** Type:
> *"Delete `TEST.DEMO` from XENO. Yes, confirm."*

**Expected:** `deleteItem` runs (with `confirm=true`), TEST.DEMO is gone. Run `itemQuery` for it to show the row doesn't return.

**Talking point:** *"Same gating for destructive ops as for writes. There isn't a special path for delete — and there can't be, because the AI never has to guess which path to take. The contract is uniform."*

## Risk acknowledgements (pre-empt before BK raises them)

| Concern | Position |
|---|---|
| "Are you bypassing my auth?" | No — Integrated Auth into SQL Server is exactly the same identity context the rest of TimeScape uses. The MCP runs in the calling user's logon session. |
| "Are you running anything inside my engine?" | No — we wrap the public `timescape.pyd` CPython binding, which is the documented external integration surface (same one your .NET API uses). No code injected into your service processes. |
| "What if the AI runs amok?" | Twelve of sixteen tools refuse on the first call and require explicit confirmation. The pattern is uniform; there's no path that skips it. |
| "Tenancy — could one customer see another's data?" | CONTRACT §0: hard rail. The MCP per-deployment serves the customer's own data only. No Xeno-internal, no cross-tenant aggregates. Foundational, in the codebase. |
| "Does this need internet?" | No. Stdio transport, in-process Python call, local SQL Server. The MCP can run on an air-gapped host. |
| "What does deinstall look like?" | `uv pip uninstall xenomorph-mcp` + remove the Claude Desktop config entry. TimeScape is untouched. |

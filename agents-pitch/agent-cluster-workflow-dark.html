ANT Agent Cluster Workflow

The "AI Agent Cluster" pattern, mapped onto ANT's real substrate. The portfolio orchestrator is the always-on Chair; product work happens in rooms; shared skills and templates live in the Obsidian vault pool. Context survives across the cluster through durable room memory, ant chair handoff, and the Cross-Agent Inbox — not through any single session.

Panel A · Cluster topology & the routing gate

Chair · Portfolio level

Strategic orchestrator & gatekeeper · always-on, cheap model
  • Watches every room, compiles per-room digests
  • Runs "ripple analysis" on shared-template changes
  • Approves promotion of room patterns → shared pool
  • ant chair enable / handoff / board

Room Agents · Product level

Deliverable owners — reports · decks · sheets
  • Own feature work, drafts & QA for one room
  • Identify reusable patterns worth promoting
  • Cannot edit the shared vault directly

Vault Agents · Shared level

Obsidian skill + template pool maintainers
  • Build room-agnostic shared skills & templates
  • Implement approved pattern extractions
  • Changes require Chair approval

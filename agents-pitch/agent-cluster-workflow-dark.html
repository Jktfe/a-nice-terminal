ANT Coordination & Routing — chairs are optional

The "AI Agent Cluster" pattern assumes a single mandatory orchestrator at the top. ANT does not. Coordination is substrate-first: rooms, asks, @-mentions, tasks and the Cross-Agent Inbox carry the work, and peer agents resolve each other's asks with no chair required. Two optional chairs sit on top — a Room Chair (in-room operator, max 1/room, has verbs) and an ANT Chair (the user's cross-room proxy, max 1/user, lives outside rooms, has speech). Either or both can be absent.

Re-grounded mapping: ANT rejects the single mandatory orchestrator. Rooms, asks, @-mentions and tasks are the coordination substrate and work with zero chairs. Room Chair ↔ an optional in-room facilitator (the cluster image's "facilitation/validation" work). ANT Chair ↔ an optional user-scope proxy that routes & disambiguates the cross-room asks queue — the natural home for a local signal router (tiny-router style) and the surface where the user interface matters most (decision cards, Yes/No, plain-English FYI alerts).

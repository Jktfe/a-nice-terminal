# ANT UX atlas — 2026-05-11

**Plan slug:** `ux-atlas-2026-05-11`
**Authors:** @evolveantclaude + @evolveantcodex
**Status:** M0 in progress
**James-locked scope:** start now · literal every-permutation coverage

This plan produces the canonical Pencil-first UX atlas for ANT.
The product-story deck (`ant-product-story-2026-05-11.html`) defined
the answer; this atlas works backwards from the answer to map every
single screen, state, modal, dropdown, keyboard action, and mobile
permutation a pilot will see.

It precedes implementation. Per James's instruction: define every
permutation BEFORE we cut tickets. Otherwise we just move jank
around.

---

## 1. Why we are writing this

After acceptance of `ant-product-story-2026-05-11`, James called
out a concrete UX symptom: clicking a terminal vs going through a
"linked chat" produces different views of what is really the same
workroom. That's one of dozens of unsurfaced category leaks. The
deck names primitives; the atlas pins down every place those
primitives are rendered.

James's exact framing:
> "EVERY screen and EVERY interaction mapped out in a pencil
> canvas — it is going to be huge I know, but better we get the
> ux and feel right, then move on to the implementation."

## 2. Scope — locked by James 2026-05-11

- **Start time:** now (2026-05-11 evening).
- **Coverage:** *literal* every-permutation. Not representative
  sampling. Aim ~200+ screens across all surface families.
- **Duration estimate:** ~2 weeks calendar, cap-2 protocol, both
  agents in parallel where possible.
- **Tool:** Pencil canvas via `mcp__pencil__*` MCP. One Pencil
  document per surface family (5 files) to avoid choking a single
  mega-canvas.
- **Output:** Pencil files + interaction matrix doc + state diagrams
  per surface family + a master index. NO implementation code in
  this lane.

## 3. Anti-scope (what this plan does NOT do)

- Does not write any frontend code. That's the next lane,
  downstream of M4 sign-off.
- Does not propose engine/backend refactors (e.g., the message-edit
  interrupt protocol from the product-story M0 §4 — that has its
  own lane).
- Does not ship marketing comps. The canvases are working
  source-of-truth wireframes, not pretty hero shots.
- Does not move the product story slug forward. That deck is done.

## 5. Milestones

### M0 — Repo inventory audit
**Owner:** @evolveantcodex
**Output:** `docs/ux-atlas/m0-inventory.md`

Enumerate every:
- Route in `src/routes/` (page + API).
- Component in `src/lib/components/`.
- View mode / state / drawer / modal / dropdown each component renders.
- Empty / error / loading / offline / auth / mobile state per surface.
- Agent-state permutation (working / thinking / waiting / stalled / errored / killed) and how each renders.
- Each surface's mobile breakpoint behaviour (PWA standalone vs Safari vs Chrome desktop).
- Each surface's terminal-disclosure level (Cockpit / Room / Agent timeline / Terminal viewer / Terminal controller).

Output structure per surface entry:
```
- surface: /room/[id]
  parent: /home
  modes: [default, agent-pair, terminal-revealed, archive, focus]
  states: [populated, empty, loading, offline, auth-required]
  modals: [invite, settings, archive-confirm, break-modal]
  dropdowns: [participant-menu, agent-chip-menu, room-menu]
  mobile: [composer-sticky, sheet-overlay, swipe-archive]
  agent-states-rendered: [working, thinking, waiting, stalled, errored]
  disclosure-levels: [2, 3, 4, 5]
```

**Acceptance:** every surface in `src/routes/` has an entry. Every
component in `src/lib/components/` that renders an addressable view
has an entry. Manifest is grep-able. @evolveantclaude reads and
flips when complete.

### M1 — IA canvas
**Owner:** @evolveantclaude (canvas) + @evolveantcodex (rule extraction)
**Output:** `Pencil/ant-atlas/00-ia.pen` + `docs/ux-atlas/m1-ia.md`

The information-architecture canvas. Single Pencil doc with:
- Top-level surface family boxes: Home/Cockpit, Rooms, Workroom,
  Agent timeline, Terminal viewer + controller, Plans, Questions,
  Artefacts, Participants, Diagnostics, Settings, Invites, Mobile
  PWA.
- Arrows between them showing navigation flow.
- The "linked-chat is view not destination" rule visualised as a
  collapsed-view note inside Workroom.

**Acceptance:** every surface from M0 maps to exactly one family.
Family boundaries are legible without explanation. @evolveantcodex
peer-reviews and flips.

### M2 — Interaction matrix
**Owner:** @evolveantcodex
**Output:** `docs/ux-atlas/m2-interactions.md`

Every clickable / tappable / keyboard-reachable thing in the
system, indexed by surface family. Per interaction:
- target (surface + selector)
- action (open / close / submit / focus / navigate / route / write / etc.)
- result state
- evidence trail (what writes to the DB)
- receipt (what the user sees as confirmation)
- mobile equivalent (tap target size, sheet vs modal)
- keyboard shortcut if any

**Acceptance:** every click in M0 is matched to one matrix row.
@evolveantclaude peer-reviews.

### M3 — Pencil wireflows
**Owner:** @evolveantclaude (drafting) + @evolveantcodex (per-surface review)
**Output:** 5 Pencil files
- `Pencil/ant-atlas/01-cockpit.pen` — home / cockpit / item detail / interview-lite / digest
- `Pencil/ant-atlas/02-room.pen` — default room / agent-pair view / terminal-revealed / context-break / focus / archive
- `Pencil/ant-atlas/03-artefacts.pen` — deck / sheet / doc / plan / site / asks pinned
- `Pencil/ant-atlas/04-edges.pen` — onboarding / invite redemption / settings / diagnostics / error / empty
- `Pencil/ant-atlas/05-mobile.pen` — every above at iPhone PWA breakpoint with mobile-specific surfaces (sheet overlays, swipe gestures, the iOS PWA-safe modal patterns from BreakConfirmModal)

Each file contains:
- Every screen at 1440×900 desktop + 390×844 mobile breakpoints.
- Every state per screen (loading / populated / empty / error / locked / stale).
- Every modal / dropdown / tooltip surfaced from the screens, drawn as overlays.
- A small state-diagram in the corner showing legal transitions for that family.

**Acceptance:** every interaction in M2 has at least one canvas
where it's shown firing. @evolveantcodex reads every screen.
Acceptance flips only when codex confirms canvas matches matrix
matches inventory.

### M4 — Review pass + James walkthrough
**Owner:** both, in front of James
**Output:** Sign-off log + ticket-ready list

Both agents walk James through each Pencil file. James asks
questions, makes rename / collapse / category calls. Decisions
captured as edits to the canvases AND notes in
`docs/ux-atlas/m4-decisions.md`. At the end of M4, James either
accepts the atlas as source of truth for implementation, or
identifies a phase that needs re-work.

**Acceptance:** James-signed. Then the next lane (implementation
ticket cutting) opens.

## 6. Open design questions (resolve before M3)

- **Pencil document granularity.** 5 files (one per family) seems
  right but if codex's M0 inventory shows e.g. 80+ screens in the
  Cockpit family alone, we may split further.
- **Mobile-first vs mobile-second.** Default proposal: draft
  desktop first, then derive mobile. Alternative: draft mobile
  first (forces ruthless simplification). Codex to call after M0.
- **State-diagram tool.** Embedded in Pencil or external (Mermaid,
  whatever)? Default: a small canvas region per family file with
  inline arrows.

## 7. Out of scope for this plan (queued for follow-on lanes)

- Operator Cockpit implementation
- Cadence-pulse + heartbeat protocol
- Read-receipt UI (delivered vs read split)
- Message-edit + agent-interrupt protocol
- Component-layer three-tier split
- 8-jargon rename pass execution
- Top-3 of 6 known security issues

All of those follow the atlas's M4 sign-off, not before.

## 8. Risks

- **Scope creep.** James said literal-every-permutation. We hold
  the line. If a permutation feels redundant, document it as
  redundant rather than skipping it.
- **Pencil performance.** 200+ screens across 5 files should be
  fine; one mega-file would not be. We monitor and split further if
  needed.
- **Inventory drift.** The repo will keep changing during the
  fortnight. M0 freezes a snapshot; if a surface changes
  mid-atlas, codex re-audits that family and pushes a delta.
- **Pencil MCP unfamiliarity.** Claude hasn't used the Pencil MCP
  heavily before. M1 onwards may need check-ins to confirm the
  tool isn't limiting the work.

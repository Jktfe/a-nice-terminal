How an ANT agent works for hours without carrying hours of context

The product move is simple: short-term context becomes disposable, while the harness makes resources, proof, and durable learning explicit. The agent does not remember everything. It knows how to work the system.

1. What changes

The agent stops treating one conversation window as the whole brain. Context is just the working bench. The durable system is the bench plan, the shelves, the labels, and the inspection record.

Today: context stuffingfragile
Long chat = fake continuity

The agent carries old turns until compaction removes detail or stale claims feel current.

Memory = one vague bucket

Semble, Mempalis, Obsidian, room history, GitHub, and screenshots blur together.

Review = last answer wins

Claims arrive faster than the proof trail, so confidence is not tied to source quality.

Target: harnessed autonomyexplicit
Fresh context starts from bearings

Every new agent reads goal, room state, open decisions, linked lanes, and verification gates before acting.

Resources are typed primitives

Search, rooms, memory, source files, web, and artefacts each have a reason to be called.

Only durable lessons survive

Evidence-backed findings are promoted; scratch notes and failed branches stay disposable.

2. The agent loop going forward

This is the cold-start ritual and work cycle for any agent brought into a long-running ANT track.

1

Get bearings

Read the room brief, current goal, linked rooms, last checkpoint, and owned lane before touching work.

2

Set a small run

Choose one bounded question or build step. Scratch context is loaded for that run only.

3

Search as code

Compose retrieval: retrieve, fanout, dedupe, rank, pack.

4

Move depth out

Use a linked room or note for noisy exploration. Keep the main room for decisions and artefact review.

5

Bank proof

Attach source links, screenshots, commands, diffs, or direct quotes before promoting the claim.

6

Promote lessons

Only stable, reusable findings go into durable memory. Everything else stays in the lane log.

7

Leave a handoff

Write what changed, what is verified, what is open, and exactly where the next agent resumes.

3. What ANT implements underneath

The implementation is not one magic memory feature. It is a set of small primitives with clear jobs and clear promotion rules.

Goal ledger

Current work state

Room-scoped goal, owner, open decisions, blocked items, acceptance gates, and latest verified checkpoint.

Resource router

Search primitive selection

Exact local lookup, Semble-style discovery, web/source fetch, GitHub stars, screenshots, or prior room history.

Lane rooms

Noise control

Deep research happens in linked rooms with their own artefacts, while Research Colony stays a review surface.

Evidence packer

Claims need backing

Promoted notes carry source, timestamp, command output, screenshot, file path, or explicit uncertainty.

Memory promotion

Disposable to durable

Mempalis/Obsidian entries are written only when a lesson is reusable and stable enough to survive compaction.

Context pack

Small working set

The next agent receives the minimum useful bundle: goal, constraints, evidence links, and next action.

5. Implementation decisions to make

These are the product choices that decide whether the system becomes useful or just another memory drawer.

Default route

  • Main room gets summaries and review links.
  • Linked rooms get raw exploration.
  • Artefacts become first-class review objects.

Promotion rule

  • No durable memory from a single unverified answer.
  • Require evidence or mark uncertainty.
  • Prefer small reusable lessons over transcripts.

Failure mode

  • Do not let memory become a dumping ground.
  • Do not let linked rooms hide decisions.
  • Do not let context packs overgrow into full history.
The working slogan: harness first, memory second. ANT wins when an agent can lose its short-term context, come back through the harness, and continue with better evidence than before.

## Observations

### Cold-start variance

First call to any endpoint after a quiet period (~30s+ idle) takes
2-6 seconds. Subsequent calls are 8-20ms. This is generic
SvelteKit/nodemon cold-load behaviour in dev mode, not endpoint-
specific:

- `/api/sessions/<id>` first hit: 3085ms; runs 2-5: 9-13ms
- `/participants?include_counts=1` first hit: 6270ms; runs 2+: 8ms

The SQL itself runs in <1ms (verified directly via sqlite3 CLI).
The cold-start cost is route-module hot-loading, not query slowness.
Production builds (`bun run build && bun run start`) should not
exhibit this since modules load once at boot.

### Bandwidth observations (unchanged from M1 audit)

- `/api/sessions` is 74KB for 86 sessions (still fat — trim opportunity)
- `/api/sessions/<id>/tasks` is 37KB (cap candidate)

Codex's eaeabaf already moved /api/sessions to setTimeout(0) deferral,
so this isn't on the critical path — but on first cold load the bundle
+ TLS + this 74KB stack up.

Review Cards

9 scoped ideas
P0 Message delete needs a guard rail chat safety

Shape

A destructive click opens a compact confirmation popover beside the message: Delete / Cancel. After delete, show a 10 second Undo row in the chat footer. Bulk or permanent deletes always require confirmation.

Acceptance

  • Accidental delete can be cancelled before it happens.
  • Deleted message can be restored without reloading.
  • Reply and delete controls are visually separated enough to avoid the original mistake.
P0 Image uploads cannot be anonymous or unlimited cost · security

Shape

Inline image rendering does not imply public upload. Default storage is local/private artifact serving. Remote blob storage is a fallback only for remote sessions that cannot directly serve local artifacts. Any remote upload requires an authenticated session, server-side authorization, short-lived upload token, image-only content types, maximum size, per-user/session quota, and rate limit. Anonymous upload routes are not allowed. Limits must be configurable policy, not hardcoded friction.

Acceptance

  • Local/private artifact serving is used by default for generated screenshots and evidence.
  • Unauthenticated remote upload request returns 401 and never receives a Blob client token.
  • Allowed uploads are limited to image MIME types and a configured maximum size.
  • Per-session quota/rate limit prevents remote storage cost abuse.
  • Changing max size, quota, allowed MIME types, or remote-storage enablement requires config/admin policy only, not a code change.
P1 Visual folder navigation that still leaves evidence terminal navigation

Shape

A project jump drawer with recent folders, pinned workspaces, fuzzy search, and a visible command preview. Selecting a folder pastes and runs cd /absolute/path in the active terminal, so the raw terminal shows exactly what changed and when.

Acceptance

  • Pick a nested repo in three keystrokes or less.
  • The terminal timeline contains the actual cd command with timestamp.
  • No silent cwd teleporting.
P1 CLI blocked-prompt visibility before you open tmux agent status

Shape

When prompt-bridge or driver fingerprinting knows a terminal is an agent CLI, expose waiting state outside the raw timeline: sidebar status dot, top-level "agents waiting" counter, and a focused-terminal summary line such as Codex is waiting for file-write approval.

Acceptance

  • Missed confirmation screen is visible from the sidebar within two seconds.
  • Status does not get written into PTY output.
  • Plain shells and dev servers do not show agent-blocked status.
P1 CLI selector becomes a compact dropdown linked chat

Shape

Replace the horizontal CLI strip with a searchable dropdown grouped by active sessions, pinned agents, and recent CLIs. Each row shows handle, model/driver, cwd, and status. The selected CLI becomes the scoped target for the compose box.

Acceptance

  • No horizontal scroll is needed to select a CLI.
  • Keyboard shortcut opens selector and filters immediately.
  • Selected CLI is always visible in the compose header.
P1 LinkedChat routing only when a CLI is selected routing rule

Shape

If a CLI target is selected, the compose box behaves like a chatroom: send message to the CLI, show the response in LinkedChat, and keep the raw terminal one click away. If no CLI is selected, direct entry stays ANT + terminal.

Acceptance

  • CLI-selected messages return in the linked chat surface.
  • No-target messages never route as chat by accident.
  • Each chat response links back to raw terminal time/session evidence.
P1 Pinned terminals live in the sidebar beside chats left rail

Shape

Let terminals be pinned into the same left sidebar used by chats. A pinned terminal row shows name, cwd, intent-like type from existing context, last activity, and a discreet status dot. No output preview in the rail.

Acceptance

  • Pin a dev server and a CLI; both survive reload.
  • One click returns to the exact terminal/session.
  • Status is useful but quiet: running, waiting, failed, listening, idle.
P1 References are captured into the right panel files · links · artifacts

Shape

When chat text contains a local file path, repo-relative path, URL, generated artifact, screenshot, or image path, ANT extracts it into the right reference panel. Group by Files, Links, Images, and Artifacts. Each reference keeps source message, agent, timestamp, and session provenance.

Acceptance

  • Posting docs/ANT-user-notes-review.html creates a clickable file reference.
  • Posting output/playwright/ant-user-notes-review.png creates both an image reference and a file reference.
  • Deleting or editing chat text does not silently orphan references; provenance degrades visibly.
P1 Images render directly in chat visual evidence

Shape

Image attachments, generated screenshots, local image paths, and trusted image URLs render as inline thumbnails in chat with open, copy path, and reveal-in-reference-rail controls. Unsafe or untrusted sources render as a placeholder that must be opened explicitly.

Acceptance

  • A Playwright screenshot path posted in chat shows a thumbnail without needing to open the file manually.
  • Inline rendering uses the artifact/file-serving layer, not raw PTY output.
  • The same image appears in the right panel with source message and timestamp.

Integrations Without Breaking The Product

A selective proposal from the integrations research: what to spike, what to learn from, and what to park until v4 is stable. The bias is to protect existing ANT work while banking the strongest ideas for native apps, plans, and agent search.

Recommendation

Do not adopt a bundle of integrations. Approve three small, reversible spikes: write an ANT threat-model doc using Shellin's structure as inspiration, test SVAR Gantt as a read-only plan cockpit layer, and test Semble as an optional MCP/code-search assistant. Everything else stays as design reference until v4 launch pressure drops.

Three Tracks

Safe sequencing

Track A: v4 Launch Safe

Low-risk work that improves the public repo or existing product surface.

  • Threat-model doc before AGPL/public push.
  • Read-only Gantt cockpit spike.
  • No schema replacement, no route rewrites.

Track B: Agent Tooling

Internal productivity improvements that can be removed if they disappoint.

  • Semble as optional MCP, not a required runtime dependency.
  • Measure whether it reduces file-reading churn.
  • Keep `rg` and direct file reads as baseline tools.

Track C: Native Premium

Strong concepts for antchat/native apps after baseline stability.

  • Shellin-style P2P terminal transport as privacy wedge.
  • Split terminal bytes from control events.
  • Park until native app and invite flow are stable.

Decision Board

Proposed picks
Spike now

ANT threat model

Use Shellin's threat-model shape as a reference: trust boundaries, what the server can see, sensitive code paths, token replay controls, and scope limits.

  • Why: public AGPL release needs trust clarity.
  • Proof: `docs/threat-model.md` reviewed before repo reset/push.
  • Kill criteria: none; this is documentation and should happen.
Tier: OSS core
Risk: low
Owner: codex/claude
Spike now

SVAR Gantt for plans

Test `@svar-ui/svelte-gantt` as a read-only render layer inside `/plans/[id]`. This is the "Gantt chat/cockpit" idea, but constrained to a proof before adoption.

  • Why: closes a real plan-cockpit gap without inventing a chart engine.
  • Proof: one real plan rendered with tasks, dates, hierarchy, and screenshot.
  • Kill criteria: bundle weight, styling conflict, or data adapter takes more than one focused slice.
Tier: OSS core
Risk: low-medium
Owner: svelte
Spike now

Semble MCP code search

Run Semble as optional local MCP for agent code search. It should help agents find exact code snippets without reading half the repo.

  • Why: lower context load and less noisy file-read exploration.
  • Proof: compare one real debugging task using `rg` only vs Semble-assisted search.
  • Kill criteria: slow indexing, poor relevance, awkward setup, or no measurable reduction in chatter.
Tier: internal tooling
Risk: low if optional
Owner: codex/ux
Park for native

Shellin-style WebRTC terminal transport

Bank the architecture: server handles auth/signaling, terminal bytes move P2P, and terminal/control channels are separate. This is compelling for native app privacy.

  • Why: credible premium privacy story for antchat/native.
  • Proof later: one desktop agent and one native client exchanging terminal data P2P.
  • Kill criteria: TURN complexity, mobile background limits, or debugging overhead outweigh privacy benefit.
Tier: premium/native
Risk: high
Owner: native lane later
Learn from

Superpowers process pattern

ANT already has plans, asks, claims, and verification discipline. Lift the useful operating patterns, not the framework as a dependency.

  • Why: reinforces ANT's own plan/ask discipline.
  • Proof: update ANT manual and agent onboarding with the process lessons that fit.
  • Kill criteria: any step that turns ANT into a wrapper around another agent framework.
Tier: process
Risk: low
Owner: coordinator
Learn from

Collab / Paperclip / n8n

Useful as reference material for canvas layouts, governance, budgeting, and workflow execution, but too broad or misaligned to adopt during v4 launch.

  • Why: they validate market demand and provide design vocabulary.
  • Proof: extract 3 UI/process patterns into ANT backlog, not code.
  • Kill criteria: any proposal that imports their architecture wholesale.
Tier: reference
Risk: medium-high if adopted
Owner: none now

Guardrails

Do not break v4

Adoption Rules

  • No dependency becomes production-critical from a spike.
  • No schema replacement to satisfy an integration.
  • No route rewrites while launch-critical ANT work is still active.
  • No premium/native work blocks OSS v4 launch.

Proof Required

  • Every spike has a screenshot or command transcript.
  • Every spike has a rollback path.
  • Every spike has a named kill criterion.
  • Every accepted idea becomes an ANT plan task before implementation.

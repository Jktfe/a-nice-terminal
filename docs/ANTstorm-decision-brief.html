Two Equal Tracks

Readable Surface, Evidence Surface

The rich layer does not replace the raw layer. It projects it, annotates it, and links back to it.

Track 1: ANT Terminal

  • Render from run_events. CommandBlock receives a structured event, not flat props.
  • Use three first block kinds. command_block, agent_prompt, artifact.
  • Artifacts via API. Images and screenshots render from /api/artifacts/, never PTY bytes.
  • Prompt overlays stay visual. They sit in the ANT Terminal layer only. Keystrokes still enter the PTY.
  • Raw is one click away. The escape hatch is always present and always authoritative.

Track 2: RAW Browser Terminal

  • xterm is live-tail. Active screen and bounded recent scrollback, not full transcript storage.
  • SQLite owns search. Deep history is virtualized and indexed, not replayed into xterm.
  • tmux is recovery/control. It can fill bounded gaps, but it is not canonical history.
  • Backpressure is explicit. Browser throttle signals pause upstream; bytes are buffered, never dropped.
  • Passthrough stays off. Inline graphics are a Phase 2 opt-in with security and scroll tests.

Implementation Order

Five Milestones With Acceptance Gates

Each milestone proves one architectural claim and has a rollback path if the claim fails.

M1

OSC 133 Shell Hooks

Command boundaries, current directory, and exits enter run_events with high-trust hook source.

Acceptance ls && false && echo ok creates exactly three command events with exits 0, 1, 0. Reconnect through tmux creates no duplicates or missing byte ranges.
M2

WebGL Renderer Flag

Load WebGL after document.fonts.ready, keep DOM default, and fall back on context loss.

Acceptance 100k-line burst has no blank canvas or glyph corruption. Screenshot diff below 0.5% excluding cursor blink.
M3

CommandBlock Visual

Sticky headers, concise blocks, copy/rerun/bookmark controls, compressed long output, status elsewhere.

Acceptance A non-technical viewer can skim a 30-minute session and explain what happened in under 30 seconds.
M4

Pi RPC Adapter

Drive Pi through --mode rpc and render prompt/tool/approval cards from structured events.

Acceptance Replaying parsed raw offsets reproduces the live event count, kinds, payload hashes, timestamp order, and raw refs.
M5

Hermes ACP Client

ANT consumes Hermes ACP with 1:1 ANT session to Hermes profile/session mapping.

Acceptance Pi RPC and Hermes ACP fixtures normalize to the same block sequence and render path. Only source/protocol payloads differ.

Deferred Bets

Explicitly Not In Phase 1

These are not forgotten. They are parked until the core contract is stable.

Inline graphics protocols

Sixel, Kitty graphics, and iTerm2 OSC 1337 stay out of Phase 1. They require tmux passthrough and could destabilize scrollback/reconnect.

ANT-as-ACP-server

Deferred until ANT-as-ACP-client with Hermes works. Do not make ANT pretend to be an editor before the UX shape is proven.

Markdown-first storage

Markdown remains a projection/export. Canonical truth is JSONL plus raw transcript chunks because that keeps structured rendering and SQL projection cheap.

Renderer or tmux replacement

Only revisit custom renderers or tmux replacement if WebGL/DOM and tmux backpressure fail concrete acceptance tests.

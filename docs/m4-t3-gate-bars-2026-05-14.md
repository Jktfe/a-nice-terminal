# M4 T3 CLI Gate Bars

Scope: read-only gate scaffold for the Remote ANT CLI slice. T3 must not claim PASS until T2/T2.5 routes are canonical PASS and every CLI wrapper below is covered by direct `bun test scripts/...test.mjs` evidence. Admin calls use `--admin-token` or `ANT_ADMIN_TOKEN`; missing admin token fails before fetch. `--json` prints the server payload unchanged. Text output must never print admin bearer values.

## Cross-Cutting Bars

- T3-C0 dispatch: `scripts/ant-cli.mjs` exposes `remote` and `remote-room` without breaking existing verbs; tests cover main-runner dispatch for both primary verbs.
- T3-C1 failure shape: every required flag is validated before fetch; non-2xx responses return non-zero with status text and redact admin token, bridge token, invite code, and remote code where present.
- T3-C2 count honesty: contract says 7 manifest rows, not 7 raw subcommands. T3 evidence must map wrappers to `remote-admit`, `remote-redeem`, `remote-mapping`, `remote-room-send`, `remote-room-status`, `remote-room-ack`, and `remote-room-quarantine`.
- T3-C3 route gaps RESOLVED via PATH A v1 narrow (T3 delta-1, 2026-05-14): `remote-room status` wraps GET /api/remote-ant/mappings/:id (mapping detail only); delivery_state counts are V2. `mapping list --include-revoked` is V2 — v1 lists active only via existing listActiveForRoom().

## T3-B1 remote-admit

- CLI: `ant remote admit --room ROOM --lifetime today|48h|7d|indefinite [--admin-token T] [--json]`.
- Route: `POST /api/remote-ant/admit` with `{roomId,lifetimePreset}` and admin bearer.
- Text: print `code`, `admission_id`, and acceptance expiry because code is returned once by design.
- Tests: happy path, `--json`, missing room, missing/invalid lifetime behavior locked, missing admin pre-fetch, 401/400 surfaced.

## T3-B2 remote-redeem

- CLI: `ant remote redeem --code CODE --admission-id ID --remote-url URL --label LABEL [--direction in|out|both] [--json]`.
- Route: POST to `${remote-url}/api/remote-ant/admissions/${id}/redeem` with `{code,remoteInstanceLabel,direction?}` and no admin bearer.
- Text: print `mapping_id` and `bridge_token` once. Do not echo `code` in errors.
- Tests: remote-url base is honoured, code/admission/label required pre-fetch, bad direction pre-fetch, 410 surfaced, token appears only on success.

## T3-B3 remote-mapping

- CLI: `ant remote mapping list --room ROOM [--admin-token T] [--json]`; `show MAPPING_ID`; `revoke MAPPING_ID`.
- Routes: `GET /api/remote-ant/mappings?roomId=...`, `GET /api/remote-ant/mappings/:id`, `POST /api/remote-ant/mappings/:id/revoke`.
- Text: list one line per mapping with id, remote label, direction, last_seen; show detail without token bytes; revoke prints revoked id.
- Tests: list requires room/admin and sends bearer, show/revoke require id, no token bytes rendered, 404/401 surfaced, main dispatch covers nested mapping subverbs.

## T3-B4 remote-room-send

- CLI: `ant remote-room send MAPPING_ID --msg TEXT [--admin-token T] [--json]`.
- Route: `POST /api/remote-ant/mappings/:id/send` with admin bearer and body `{kind:"message",payloadJson,replaySignature?}`.
- Text: print event id, status, delivery_state. `--msg` must be encoded into payloadJson deterministically.
- Tests: missing id/msg/admin pre-fetch, correct route/body/header, 201 text/json, 400/401/404/413 surfaced, no bridge bearer accepted by wrapper.

## T3-B5 remote-room-status

- CLI: `ant remote-room status MAPPING_ID [--admin-token T] [--json]`.
- Route LOCKED v1: wraps GET /api/remote-ant/mappings/:id (PATH A amendment). Counts are V2.
- Text: print mapping id + label + last_seen + revoked.
- Tests: missing id/admin pre-fetch, GET against mapping detail, JSON pass-through, 404/401 surfaced.

## T3-B6 remote-room-ack

- CLI: `ant remote-room ack EVENT_ID [--admin-token T] [--json]`.
- Route: `POST /api/remote-ant/quarantine` with `{eventId}` and admin bearer.
- Text: print acked event id.
- Tests: missing event/admin pre-fetch, correct body/header, 200 text/json, 404 already-acked/unknown and 401 surfaced.

## T3-B7 remote-room-quarantine

- CLI: `ant remote-room quarantine list [--mapping-id MAPPING_ID] [--admin-token T] [--json]`.
- Route: `GET /api/remote-ant/quarantine[?mappingId=...]` with admin bearer.
- Text: one line per quarantined event with id, mapping_id, direction, kind, status_reason, created_at.
- Tests: admin required pre-fetch, optional mapping filter encoded, no payload over-rendering, empty list render, 401 surfaced, main dispatch covers nested quarantine list.

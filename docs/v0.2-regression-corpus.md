# ANT v0.2 Regression Corpus — Index

Human-readable index of the 9 cases that gate the v0.2 schema cut-over.
Every case is paired with a real assertion suite in
`scripts/v0.2-regression.test.ts` (30 tests across 9 cases). All 9 cases
are GREEN against the v0.2 substrate now live on main — the failure
classes captured here are structurally impossible on the v0.2 schema.

History: the original corpus shipped in closed PR #110, branched off
`feat/v0.2-schema-tables`. When the cut-over went incremental (PR #103
schema tables → #107 stores → #108 M9b identity flip → #111 M9c
chat-room dual-write, plus PR #99 identity-keys + recovery-grants),
PR #110 was closed and retargeted onto main as this current file. The
9 cases are unchanged; the impossibility framing is unchanged; cases
9b/9c that were blocked in #110 (paper-key recovery) now run green
because PR #99's primitives are live on main.

## How to read this table

| Column | Meaning |
|---|---|
| `#` | Case number — matches the stub order in the test file. |
| `Title` | Short summary of the failure mode. |
| `Source incident` | Date + room + msg_id that produced the case, plus a one-sentence description. |
| `v0.2 invariant` | The schema property that makes the failure structurally impossible (citing the concept doc). |
| `Status` | `todo` (stub only) / `asserted` (body written, awaits v0.2) / `passing` (running green against v0.2). |

2. **Assertion lands** (closed PR #110, retargeted 2026-05-30) — `it.todo()` becomes a real `it(...)` body asserting structural impossibility against the v0.2 substrate primitives.

## Permanent CI gate proposal

JWPK ratification pending. Proposal: every schema-touching PR runs this corpus
as a required check after v0.2 cut-over. Rationale: the failure modes captured
here are correlated — a single careless migration could quietly reintroduce
the `(agent_id) WHERE status='live'` partial-index assumption, for example —
so we want continuous coverage, not a one-shot gate.
